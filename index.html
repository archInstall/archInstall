Quick-Start

Simply load the newest version from: archInstall

>>> wget \
    https://raw.github.com/archInstall/archInstall/master/archInstall.bash \
    -O archInstall.bash && chmod +x archInstall.bash

archInstall ist das zentrale Module um eine Reihe von Aufgaben im Zuge der Vorbereitung, Installation, Konfiguration oder Manipulieren eines "Linux from Scratch" durchzufhren.

Das Module kann im einfachsten Fall (z.B. von einer beliebigen Life Linux Umgebung) gestartet werden.

>>> ./archInstall.bash

In diesem Fall werden alle bentigten Informationen zur Einrichtung (nur Hostname) des Systems vor Beginn des Installations Prozesses interaktiv abgefragt. Zu Beachten ist: ohne zustzliche Parameter gibt das Module keinen Feedback ber den aktuellen Zustand des Installations Vorgangs. Mit:

>>> ./archInstall.bash --verbose

bekommt man einen etwas geschwtzigeren Installationsvorgang. Mit:

>>> ./archInstall.bash --verbose --debug

werden alle Ausgaben jeglicher verwendeten Subprogramme mit ausgegeben.

Alle Parameter wie Installations haben Standardwerte. So wird im obigen Fall einfach in das aktuelle Verzeichnis ein Ordner mit dem Programmnamen erstellt und darin das Betriebssystem installiert (./archInstall/). ill man lieber eine unbeaufsichtigte Installation:

>>> ./archInstall.bash --host-name testSystem --auto-partitioning

Install to blockdevice

Im typischen Fall will man von einer Life-CD booten um das System auf einer Festplatte oder Partition zu installieren. Hierbei mssen folgende Aufgaben erfllt werden.

  • Einrichtung einer Internet Verbindung (siehe auch Abschnitt "Offline Installation")
  • Partitionierung der Ziel Festplatte
  • Formatierung der Ziel Partitionen
  • Aufbauen der Dateisystemstruktur
  • Konfiguration des Betriebssystems
  • Installation und Einrichtung eines Boot-Loaders

Alle Aufgaben bis auf die Einrichtung der Internet Verbindung (wird in der Regel von der Host Umgebung geregelt) knne mit archInstall automatisiert durchgefhrt werden. Will man z.B auf das Block Device "/dev/sdb" installieren und sich nicht selber um die Partitionierung kmmern und den kompletten verfgbaren Platz fr das Haupt System verwenden (es soll also keine Swap- oder Daten Partition erstellt werden). sieht das z.B. so aus:

>>> ./archInstall.bash --output-system /dev/sdb --auto-partitioning

Auf diese Weise wird eine uefi Boot-Partition mit 512 MegaByte eingerichtet. Der restliche Platz wird fr die Systempartition eingesetzt. Sind noch weitere Partitionen gewnscht kann man diese whrend der Installation durch weglassen des entsprechenden Parameters selber konfigurieren. Die erste Partition wird dann als Boot-Partition und die Zweite als Systempartition betrachtet. Weitere Partitionen werden ignoriert. Manuelle Partitionierung:

>>> ./archInstall.bash --output-system /dev/sdb

An dieser Stelle sei noch erwhnt, dass archInstall alle erstellten Partition automatisch mit Labels in der Partitionstabelle und auf der Partition selbst versieht. Um dieses Verhalten zu individualisieren einfach folgende Optionen nutzen:

>>> ./archInstall.bash --boot-partition-label uefiBoot \
    --system-partition-label system

Install to partition

Um z.B. aus einem Produktivsystem heraus eine alternative Linux Distribution auf eine weitere Partition zu installieren kann einfach folgender Befehl verwendet werden:

>>> ./archInstall.bash --output-system /dev/sdb2

Hier wird auf die zweite Partition des zweiten Block Devices installiert. archInstall versucht bei der Installation auf ein Blockdevice stets einen entsprechenden Uefi-Boot-Eintrag fr den Kernel mit einem Standard Initramfs und einem Ausweich-Initiramfs zu konfigurieren. Die folgenden Parameter definieren dessen Label:

>>> ./archInstall.bash --output-system /dev/sdb2 --boot-entry-label archLinux \
    --fallback-boot-entry-label archLinuxFallback

Install to folder

Um archInstall fr komplexere Szenarien zu verwenden oder nachtrgliche Manipulationen vorzunehmen ist es sinnvoll zunchst in einen Ordner zu installieren. Siehe hierzu "archInstall with Decorator Pattern", "makeXBMCLinux", "makeRamOnlyLinux" oder "makeSquashLinux" bzw. das Projekt "archInstallWrapperTemplate".

Dieser Befehl installiert ein vollstndiges System in den eigenen Home-Ordner "test" (siehe auch Installation ohne root Rechte).

Automatic configuration

archInstall konfiguriert das neu eingerichtete System vollautomatisch. Folgende Tasks wurden automatisiert:

  • Tastaturlayout einstellen
  • Einrichten der richtigen Zeit Zone
  • Setzen des Hostnames
  • Setzen des default root Passworts nach "root"
  • Erstellen eines Benutzers bzw. Benutzerordner. Das Passwort wird initial wie der Name des Benutzers gesetzt.
  • dhcp Dienst fr alle Netzwerk-Interfaces einrichten (siehe automatisches Einrichten von Diensten)
  • Installation der Basis Programme (siehe automatische Installation von Programmen)
  • Einrichten der Signaturen fr den Paketmanager "Pacman", um vertrauenswrdige Pakete erhalten zu knnen.
  • Einrichten aller in der nhe liegenden Server um schnelle Packet Updates und Paketinstallationen zu gewhrleisten.
  • Einrichten der richtigen Paketquellen abhngig von der aktuellen CPU Architektur.

Will man hierauf selber Einfluss nehmen, gibt es folgende Mglichkeiten:

>>> ./archInstall.bash --host-name test --user-names test \
    --cpu-architecture x86_64 --local-time /Europe/London \
    --keyboard-layout de-latin1 --country-with-mirrors Germany \
    --prevent-using-pacstrap --additional-packages python vim \
    --needed-services sshd dhcpcd apache

Um die einzelnen Konfigurationsparameter zu verstehen empfiehlt sich ein Blick auf:

>>> ./archInstall.bash --help

zu werfen.

Options

archInstall stellt ein Alphabet voller Optionen zur Verfgung. Whrend bisher zum einfachen Verstndnis immer sog. Long-Options verwendet wurden, gibt es fr jede Option auch einen Shortcut.

>>> ./archInstall.bash --user-names mustermann --host-name lfs

ist quivalent zu:

>>> ./archInstall.bash -u mustermann -n lfs

Alle Optionen bis auf "--host-name" und "--auto-partitioning" haben Standardwerte. Diese beiden werden sofern nicht von vorne herein angegeben interaktiv abgefragt. Alle Standardwert knnen mit Hilfe von:

>>> ./archInstall.bash -h

oder

>>> ./archInstall.bash --help

oder

>>> ./archInstall.bash --keyboard-layout de-latin1 -h

eingesehen werden. Letzteres macht Sinn, da sich Standardwerte aufgrund schon ermittelten Informationen verndern knnen. So wird der Standardwert von "--key-map-configuration="KEYMAP=de-latin1\nFONT=Lat2-Terminus16\nFONT_MAP=" nach Eingabe von

>>> ./archInstall.bash --keyboard-layout us

zu: "--key-map-configuration="KEYMAP=us\nFONT=Lat2-Terminus16\nFONT_MAP=".

Der Standardwert von "--cpu-architecture" entspricht beispielsweise immer der Architektur des aktuellen Systems, um Konfigurationsaufwand zu minimieren.

Man kann Optionen die mehrere Werte annehmen auch mehrfach referenzieren. So hat:

>>> ./archInstall.bash --additional-packages ssh --additional-packages vim -f python

den gleichen Effekt wie:

>>> ./archInstall.bash --additional-packages ssh vim python

Dies ist im Decorator-Pattern ntzlich. Bei einem Doppelt referenzierten Wert berschreiben sptere Werte zuvor Definierte. Folgendes:

>>> ./archInstall.bash --host-name A --host-name B

entspricht:

>>> ./archInstall.bash --host-name B

Install offline

archInstall erstellt bei jeder Installation automatisch einen Paket-Cache, um weitere Installationen zu beschleunigen. Ist dieser einmal erstellt oder wird dieser zusammen mit dem archInstall (z.b. auf einem usb-stick) ausgeliefert kann Offline installiert werden.

Selbst wenn mit einem bereits vorhandenem pacstrap installiert wird, wird dieser temporr kopiert, gepatched und anschlieend offlinefhig ausgefhrt!

Bei Offline Installation mssen natrlich alle zustzlich ausgewhlten Pakete im Package Cache vorhanden sein. Ist dies nicht der Fall wird archInstall versuchen diese nach zu laden und im Offline-Fall einen Fehler zurckgeben.

Install without having root permissions

Prinzipiell ist es sogar mglich auch ohne root Rechte ein System aufzusetzen. Hierbei werden jedoch folgende Einschnitte gemacht:

  • Die Programm "fakeroot" und "fakechroot" mssen zustzlich installiert sein.
  • Ein bereits installiertes Pacstrap kann nicht eingesetzt werden
  • Zustzlich erstellten Benutzer kann whrend der Installation kein automatisch erstellter Home-Ordner geliefert werden, da die Rechte oder root nicht richtig gesetzt werden knnten.
  • Es kann nur in "Ordner" (bzw. siehe nchsten Punkt) installiert werden.
  • Das System wird in ein tar-Archiv ohne Speicherung entsprechender Datei Rechte Attribute gepackt.
  • Das Tar Archiv muss als "root" entpackt werden bevor das Ergebnis verwendet oder fehlerfrei gebootet werden kann.

Useful tips and debugging informations

Whrend der Entwicklung haben sich eine Reihe von Optionen bewhrt um Fehler bei der Entwicklung von Wrappern zu finden.

Die Option "--prevent-using-pacstrap" oder "-p" verhindert ein bereits installierten Pacman fr die Installation zu verwenden. Dies ist notwendig wenn man sein Pacman so konfiguriert hat, das z.B. Pakete wie der Kernel oder Pacman von manipulierten User Repositories abhngen. Mit "--prevent-using-pacstrap" wird eine neue Version von Pacman in einer Change-Root-Umgebung ausgefhrt.

"--prevent-using-native-arch-chroot" oder "-y" ist sinnvoll wenn man Indexing Dienste wie Ubuntu's "Zeitgeist" oder "Dropbox" verwendet, die das Unmounten von Mountpoints whrend der Installation verhindern, da sie auf diesen noch lesen/schreiben.

Installiert man von einer Live-CD auf ein Block Device bootet das System nach erfolgreicher Installation automatisch in das neu generierte System. Will man noch etwas nachbessern oder berprfen, bietet sich die selbsterklrende Option "--no-reboot" bzw. "-r" an.

Mchte man die Pakete "base-devel", "sudo" und "python" haben, geht das mit dem Shortcut: "--install-common-additional-packages" oder "-z".

Will man eine beliebige Liste von Paketen integrieren:

>>> ./archInstall.bash --additional-packages ssh python2 vim

Sollen Dienste schon beim ersten Start automatisch gestartet werden:

>>> ./archInstall.bash --needed-services sshd dhcpcd

Um die Installation zu beschleunigen kann auf ein zentral verwalteten Paket Cache verwiesen werden:

>>> ./archInstall.bash --package-cache-path /var/cache/pacman/pkg/

Extensions

Eine Menge von Werkzeugen um diverse ArchLinux Setups fr verschiedene Anwendungsflle zu bauen.

makeRamOnlyLinux

makeRamOnlyLinux.bash generiert ein ArchLinux-System (oder ein beliebiges auf ArchLinux basierendes System, welches mit einem Script generiert werden kann, welches Interface und API von archInstall bereitstellt) und packt es in eine bootbare initramfs und stellt einen passenden Kernel dazu bereit.

Interface

Das Skript bentigt als Argument den Namen der zu erstellenden Initramfs-Datei. Der passende Kernel wird an gleicher Stelle nur dem Suffix 'Kernel' erstellt. Alle Optionen von archInstall werden untersttzt, es gibt keine zustzlichen Optionen. Neben der obligatorischen archInstall.bash bentigt dieses Skript packcpio.sh, welches im selben Ordner liegen muss.

Example

Der folgende Befehl erzeugt ein Basis-ArchLinux-System in ein initramfs namens "baseArchLinux.img" und kopiert einen dazu kompatiblen Kernel nach "baseArchLinux.imgKernel".

>>> ./makeRamOnlyLinux baseArchLinux.img

makeSquashLinux

MakeSquashLinux generiert ein ArchLinux-System (oder ein beliebiges auf ArchLinux basierendes System, welches mit einem Script generiert werden kann, welches Interface und API von archInstall bereitstellt), patched die Konfiguration so, dass das booten aus dem Netzwerk ermglicht wird und packt dieses als squash-Dateisystem. Zustzlich wird eine Netzwerk-Boot-fhige initramfs erzeugt und ein fr aufs gepatchter Kernel aus den Paket-Quellen kopiert.

MakeSquashLinux kann nicht von Wrappern verwendet werden, welche nach dem generieren des Grundsystems weitere Anpassungen vornehmen mchten. Da der Download der squash-Datei ein paar Sekunden in Anspruch nimmt, wird beim booten per 'plymouth' ein splash-screen angezeigt.

Interface

Usage: ./makeSquashLinux.bash <squashfsFilePath> <kernelFilePath> <initramfsFilePath> [options]

makeSquashLinux installs an arch linux into a squashfs file.

Option descriptions:
    -W --wrapper <file>  Use wrapper in <file> to generate the root-Filesystem
    -X --xbmc Use "../makeXBMCLinux/makeXBMCLinux.bash" as wrapper.

Wrapper sind hier Skripte, welche den Standard-Setup von archInstall durch weitere Konfiguration oder Programme ergnzen (makeSquashLinux ist also selbst auch ein Wrapper).

Darberhinaus knnen alle Optionen von archInstall verwendet werden. Als Pfade werden lokale Dateinamen und scp-kompatible Pfade akzeptiert. So ist es mglich die einzelnen Dateien auf verschiedenen Rechnern zu speichern (mit den intitramfs-Optionen ist es sogar mglich das root-squashfs aus einem mit dem Ort des Kernels oder tftp-Servers nicht verbundenen Netz zu laden).

scp-Pfade haben die Form "user@ssh-server:/pfad/zu/datei"

Initramfs parameter

Um ber das Netzwerk booten zu knnen mssen dem initramfs auf der Kernel-Kommandozeile einige Informationen mitgegeben werden:

  • Art der Netzwerkverbindung: ip=<client-ip>::<gw-ip>:<netmask>:<hostname>:<device>:<autoconf> Alle Parameter sind optional. <client-ip>: Statische IP-Adresse, die angefordert werden soll. <gw-ip>: IP-Adresse des Gateways. <netmask>: Netsmaske. Wenn leer wird diese aus der client-ip generiert und/oder durch die Antwort von DHCP/BOOTP berschrieben. <hostname>: Wenn leer wird die ASCI-codierte IP-Adresse verwendet. Kann durch DHCP-Antwort berschrieben werden. <device>: Netzwerkkarte, die verwendet werden soll (ermglicht absurde Use-Cases). Wenn leer wird die erste Karte verwendet, auf der eine Antwort kommt. <autoconf>: Methode, die fr den Verbindungsaufbau genutzt werden soll. Mgliche Parameter sind rarp, bootp, dhcp, all oder static. Wenn dieses Feld leer ist wird all verwendet, also alle Protokolle durchprobiert. Bei static kommt keines der Protokolle zum Einsatz und es wird eine Verbindung anhand der vorherigen Angaben hergestellt. Beispiel: ip=::::::

  • Ort des root-Dateisystems: Der 'url'-Parameter berschreibt 'nfsroot', unabhngig von der Reihenfolge.

    • Bei squashfs werden URLs mit den Protokollen http, https und ftp untersttzt (eine DNS-Auflsung findet nicht statt): url="<protocol>://<server-ip>[:<port>]/<path-to-file>" <protocol>: Es werden http, https und ftp untersttzt. <server-ip>: IP-Adresse des Hosts, auf dem die squashfs-Datei liegt. [:<port>]: Optional, Port auf dem der Host mit dem Angegebenen Protokoll erreicht werden kann. <path-to-file>: Pfad zur squashfs-Datei.
    • Bei nfs: nfsroot=[<server-ip>:]<root-dir>[,<nfs-options>] Beispiel: url="http://192.0.43.10/example.squashfs"

makeUnityLinux

Installiert ein Arch Linux mit lightdm als Display-Manager und Unity als Window-Manager/Arbeitsumgebung. Es werde auch andere Ubuntu-Typische Pakete, wie Libre-Office oder Thunderbird installiert.

Interface

Das Interface ist mit dem von installArchLiunx.bash identisch, zustzlich kann mit der Option

-W --wrapper

makeXBMCLinux

makeXBMCLinux ist ein Wrapper-Skript fr archInstall, welches das gleiche Interface untersttzt. Neben dem Grundsystem wird dabei eine XBMC-Umgebung installiert, welche beim booten automatisch startet.

Interface

Alle Optionen von archInstall werden untersttzt, allerdings werden alle ntigen Angaben mit Standard-Werten belegt, das Skript luft also immer ohne Nutzer-Interaktion durch. Wenn kein Output-System ('--output-system') angegeben wird, wird das System in einen Ordner im aktuellen Verzeichnis installiert (von wo man es dann z.B. per rsync an den Zielort kopieren kann). Das Skript kann z.B. von makeSquashLinux oder makeRamOnlyLinux eingebunden werden.

Options

  • -R --ram-only Build an initramfs-file named 'filename'.

  • -M --media-files Give a folder with media-files to copy into the system. This option doesn't work in combination with "--ram-only". (Hint: makeRamOnlyLinux.bash --wrapper makeXBMCLinux.bash --media-files)

  • -W --wrapper Use wrapper in to generate the root-filesystem. This option doesn't work in combination with "--ram-only".

Use cases

Netzwerkboot-fhiges squashfs samt kernel und initramfs mit XBMC-Frontend und Media-Dateien bauen:

./makeSquashLinux.bash root.squash kernel initramfs --wrapper \
    ../makeXBMCLinux/makeXBMCLinux.bash --media-files ~/Musik/Justin_Bieber

XBMC auf der lokalen Partition (/dev/sdz1) installieren:

./makeXBMCLinux/makeXBMCLinux.bash -o /dev/sdz1

XBMC auf der lokalen Festplatte (/dev/sdz) installieren:

./makeXBMCLinux/makeXBMCLinux.bash -o /dev/sdz
